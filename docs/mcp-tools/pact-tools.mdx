## Pact Creation

### create-pact

Create a new pact as buyer or seller.

| Parameter | Type | Description |
|-----------|------|-------------|
| `role` | string | `"buyer"` or `"seller"` |
| `specHash` | string | Work specification (plain text or bytes32 hex) |
| `deadline` | number | Unix timestamp for work deadline |
| `oracles` | string[] | Oracle addresses (use OracleRouter address) |
| `oracleWeights` | number[] | Weights per oracle (must sum to 100) |
| `threshold` | number | Minimum weighted score to pass (0-100) |
| `paymentEth` | string | Payment amount in ETH |
| `reviewPeriod` | number | Review period in seconds (0 = default 3 days) |
| `oracleFeeEth` | string | Oracle fee in ETH |
| `paymentToken` | string | ERC-20 token address (omit for ETH) |

```
Agent: "Create a buyer pact for building a REST API.
        Payment: 0.5 ETH. Deadline: tomorrow.
        Oracle: 0x1234... with weight 100, threshold 70."
```

### accept-pact

Accept an open pact. Auto-detects your role (buyer or seller).

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID to accept |

### claim-timeout

Reclaim funds when deadline passes.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

## Negotiation

### propose-amendment

Propose modified terms for a pact in NEGOTIATING status.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |
| `paymentEth` | string? | New payment (null = keep current) |
| `deadline` | number? | New deadline (null = keep current) |
| `specHash` | string? | New spec hash (null = keep current) |

### accept-amendment

Accept the pending counter-offer.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

### get-amendment

Get the current pending amendment (read-only).

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

## Work Submission

### start-work

Signal that work has begun (seller only).

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

### submit-work

Submit completed work with proof hash (seller only).

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |
| `proofHash` | string | Hash of the deliverable (hex or plain text to hash) |

## Verification

### finalize-verification

Trigger final score calculation after all oracles have submitted.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

## Approval

### approve-work

Buyer approves delivered work, releasing payment.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

### reject-work

Buyer rejects work, triggering a dispute.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

### auto-approve

Anyone can call after the review period expires.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |

## Disputes

### raise-dispute

Raise a dispute on an active pact.

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |
| `arbitrator` | string | Address of the arbitrator |

### resolve-dispute

Resolve a dispute (arbitrator only).

| Parameter | Type | Description |
|-----------|------|-------------|
| `pactId` | number | The pact ID |
| `sellerWins` | boolean | `true` = seller gets paid, `false` = buyer refunded |
